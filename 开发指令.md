### 1. 技术栈建议
为了最完美地实现“点击课文单词触发侧边栏”的交互效果，请使用以下技术栈：
- 后端：Python (FastAPI)
- 前端：现代前端页面（HTML5 + Tailwind CSS + Vanilla JS / Vue.js），通过 Web 视图呈现。可以用 Python 统一打包或运行。
- 数据存储：开始时使用本地 JSON 文件存储课文、词汇表和练习题，方便后续扩展。

### 2. 界面布局与交互设计（参考“雅思哥”风格）
- **左侧/中部（主视图）**：课文阅读区。
  - 顶部：课文朗读控制栏（播放/暂停/进度）。
  - 主体：课文文本。文本必须经过中文分词（如使用 jieba 库），每个词语/生词都被包裹在 `<span>` 标签中。
  - 交互：用户鼠标悬停或点击任意词语时，该词语高亮，并立即触发右侧边栏的内容更新。
- **右侧（动态边栏）**：内容展示与练习区。
  - 当点击某个词语时，这里依次展示：
    1. 【词汇详情】：拼音、中英双语释义、**当前语境下的特定解释**。
    2. 【例句与搭配】：该词在真实语境中的位置、常用高频词汇搭配（Collocations）。
    3. 【语法点说明】：如果该词包含重点句式或复杂结构，显示结构解构。
    4. 【即时小练习】：紧扣该词的 3 道小题（选词填空/搭配选择/判断/改写），做完立刻给出反馈，实现从“看懂”到“会用”的闭环。
    5. 【智能智能问答】：一个内置的 AI 对话框，学生如果对这个词或句子还有疑问，可以直接在这里向大模型提问。

### 3. 核心功能模块开发计划（请分步引导我完成）
- **第一步：基础数据结构定义**。设计一个能够完美承载课文、分词索引、生词释义、例句、语法点和即时练习题的 JSON 格式。
- **第二步：后端构建 (FastAPI)**。编写 API 路由，用于读取 JSON 数据，并提供给前端；集成 Python 语音合成（如 edge-tts 或 gTTS 库）来实现高保真课文朗读音频的生成。
- **第二步：前端页面与分词点击交互**。实现双栏排版。利用前端 JS 捕获点击的词语，通过 API 实时请求该词语的属性，动态更新右侧边栏。
- **第四步：即时练习与 AI 问答机制**。编写练习题的交互逻辑（单选/填空判定）；并在右侧集成 AI Chat 窗口。
